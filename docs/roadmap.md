# Roadmap

本路线图用于约束下一阶段开发范围。当前项目先专注南京林业大学课程论文模板，保持模板轻量、稳定、易上手。

## Current Version

`v0.1.1` 是课程论文模板的当前稳定版本，目标是让用户可以直接下载、编译、改信息、写正文。

已完成：

- 课程论文模板定位。
- 根目录示例与 `templates/njfu-course-paper` 模板。
- `templates/personal-quickstart` 快速开写模板。
- XeLaTeX 本地编译、CI 编译、发布打包流程。
- FAQ、故障排查、发布流程文档。

## v0.2.0 Priorities

`v0.2.0` 优先提升课程论文写作体验，不扩大模板类型。

1. 降低新手改动成本。
   - 收敛 `main.tex` 中需要用户修改的字段。
   - 给常用字段补更清晰的占位示例。
   - 减少正文示例里不必要的复杂结构。

2. 强化模板一致性。
   - 建立根目录样式与 `templates/*/NJFUReport.sty` 的同步检查。
   - 固定模板目录内的最小可编译文件集合。
   - 检查封面、摘要、目录、正文、参考文献、附录的默认排版一致性。

3. 优化 Overleaf 体验。
   - 让 release 中的 Overleaf zip 保持最小、清爽、可直接上传。
   - 在 README 中把 Overleaf 下载入口指向稳定 release 资产。
   - 补充 Overleaf 常见编译失败的处理路径。

4. 补齐示例质量。
   - 将示例正文改成更贴近课程论文的结构。
   - 精简过度展示宏包能力的段落。
   - 保留数学、图表、代码、参考文献的基础示例。

## Later Candidates

这些内容暂不进入 `v0.2.0`，只作为后续候选：

- 学院或课程方向的封面微调选项。
- 更完整的参考文献样式说明。
- 自动截图或 PDF 预览检查。
- 模板 gallery 和截图展示。
- 更严格的 lint 与 release 校验脚本。

## Version Policy

- `v0.1.x`：修复文档、CI、打包、拼写和兼容性问题。
- `v0.2.0`：课程论文模板体验升级。
- `v0.3.0`：围绕真实使用反馈继续打磨格式、文档和发布流程。
- `v1.0.0`：当课程论文模板 API、目录结构和发布流程稳定后再考虑。
