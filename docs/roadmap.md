# Roadmap

本路线图用于约束下一阶段开发范围。当前项目先专注南京林业大学课程论文模板，保持模板轻量、稳定、易上手。

## Current Version

`v2.0.0` 是课程论文模板的当前稳定版本，主线是把普通用户要编辑的内容从模板入口文件中拆出来，让 Overleaf 包更适合零基础同学使用。

已完成：

- 课程论文模板定位。
- 核心样式宏包 NJFUReport.sty（三线表、算法、定理、盲审、英文摘要、TikZ 等）。
- 根目录示例与 `templates/njfu-course-paper` 模板。
- `templates/personal-quickstart` 快速开写模板。
- 参考常见南林课程论文版式的封面和目录样式。
- XeLaTeX 本地编译、CI 编译、发布打包流程。
- CI 中的 release 打包与资产洁净度校验。
- CI 中的模板结构检查和发布资产清单校验。
- 面向零基础用户的 Overleaf 下载入口、学生快启、格式边界和提交前检查清单。
- FAQ、故障排查、发布流程文档。

## Completed in v2.0.0

v2.0.0 已完成课程论文模板的用户接口重构。这个版本包含不兼容变更，但变更目标只服务于降低零基础使用门槛。

已完成：

- 稳定 v2 文件结构。
   - `paper-info.tex` 只放题目、课程、学院、姓名、学号、教师和盲审开关。
   - `main.tex` 只做编译入口，不再要求普通用户编辑。
   - `sections/paper.tex` 作为正文入口，避免 `1-example.tex` 给用户造成“示例文件可以删掉”的误解。
- 完成 v1.x 到 v2.0.0 迁移说明。
   - 明确从旧 `main.tex` 复制基本信息到 `paper-info.tex`。
   - 明确从旧 `sections/1-example.tex` 复制正文到 `sections/paper.tex`。
   - Release 说明必须提示 v2.0.0 是不兼容升级。
- 新增 CI 检查，防止新手文档重新指向旧入口。

## Next Priorities

下一阶段优先做真实用户验证和小范围体验修正，避免在模板已经可用后继续增加复杂度。

1. 做零基础用户试用。
   - 让未使用过 LaTeX 的同学从 Release 下载 Overleaf zip。
   - 观察是否能独立完成改信息、写正文、插参考文献、下载 PDF。
   - 只记录真实卡点，不预先增加新功能。

2. 继续观察格式需求。
   - 如果多个课程都反馈同类格式问题，再进入模板样式调整。
   - 不为单个课程样例引入破坏通用性的默认设置。

3. 维护发布质量。
   - 每次发布继续保留 4 个资产。
   - 每次发布前运行编译、模板结构检查和 release 资产校验。
   - Release 说明继续明确普通同学应下载 `*-overleaf.zip`。

## Later Candidates

这些内容暂不进入下一阶段主线，只作为后续候选：

- 学院或课程方向的封面微调选项。
- 更完整的参考文献样式说明。
- 自动截图或 PDF 预览检查。
- 模板 gallery 和截图展示。
- 更严格的 lint 规则。

## Version Policy

- `v1.0.x`：修复文档、CI、打包、拼写和兼容性问题。
- `v1.1.0`：课程论文模板体验升级。
- `v1.2.0`：工程质量和发布可靠性专项，确保普通用户拿到的 Overleaf 包干净、可编译、选择成本低。
- `v2.0.0`：模板用户接口和 Overleaf 包文件结构的不兼容升级，目标是降低零基础用户误改模板结构的概率。
